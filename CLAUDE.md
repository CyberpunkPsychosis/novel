# 小说创作协作规则

这是一个人机协作写小说的仓库。作者（用户）出意图，AI 出文字。AI 在本仓库工作时必须遵守以下流程，**不得跳过**。

## 写作前（每次写正文之前）

按顺序读取以下文件，作为本次写作的上下文：

1. `0-故事核/故事核.md` —— 确认主题和整体方向
2. `1-设定库/角色卡/` 中本章出场角色的角色卡
3. `2-大纲/章节大纲/` 中本章的章纲（如果没有，先和作者一起写章纲，再写正文）
4. `3-文风/文风指南.md` 和 `3-文风/样章.md` —— 对齐文风
5. `5-一致性/状态追踪.md` —— 确认角色当前状态、未回收的伏笔、时间线位置
6. 前一章正文的结尾部分 —— 保证衔接

## 写作中

- 正文写入 `4-正文/第NNN章-章节名.md`，章节号三位数补零
- 严格遵守章纲的剧情节拍；想偏离章纲时，先停下来和作者商量，不要自作主张
- 角色的言行必须符合角色卡；如果剧情需要角色"成长/变化"，变化本身要在章纲里有依据
- 不引入设定库里没有的重要设定（新地点、新规则、新组织）；确需新增时，先补设定库再写正文

## 写作后（每写完一章必须做）

1. 更新 `5-一致性/状态追踪.md`：
   - 角色状态表（位置、伤势、持有物、知情范围、关系变化）
   - 伏笔账本（本章埋设的新伏笔、回收的旧伏笔）
   - 时间线（本章覆盖的故事内时间）
2. 如果本章对角色有重要塑造，回写到对应角色卡的「经历更新」一节
3. 对照 `5-一致性/检查清单.md` 自查一遍

## 协作风格

- 全程使用中文
- 作者说"写下一章"时，先给出本章的节拍简述（3-5 行）让作者确认，再动笔写全文
- 单章正文一次性写完，不要分段询问
- 修改意见直接改正文，大改（推翻半章以上）前先确认
- 每完成一章，提交一次 git commit，commit message 格式：`第NNN章：章节名`

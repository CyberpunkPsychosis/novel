# 小说创作工作流

人机协作写长篇小说的仓库。结构针对 AI 长篇写作的三大痛点设计：**人设漂移、伏笔丢失、文风不稳**。

## 目录结构

```
0-故事核/      故事的一句话核心、主题、目标读者 —— 一切决策的最高依据
1-设定库/      世界观、角色卡、时间线 —— 防人设漂移
2-大纲/        全书大纲 + 逐章章纲 —— 防剧情崩坏
3-文风/        文风指南 + 定稿样章 —— 防文风不稳
4-正文/        每章一个文件
5-一致性/      状态追踪（角色状态 + 伏笔账本）+ 检查清单 —— 防伏笔丢失
```

## 使用方法

1. **开新书**：和 AI 对话填好 `0-故事核/故事核.md`，再逐步充实设定库和全书大纲
2. **写一章**：说"写第 N 章"，AI 会先读档案、给出节拍简述、确认后写全文
3. **改稿**：直接说修改意见，AI 改完会同步更新一致性档案
4. **防崩坏的关键**：`5-一致性/状态追踪.md` 是全书的"存档点"，AI 每章写完必须更新它

协作规则的完整定义见 `CLAUDE.md`，AI 在本仓库工作时自动遵守。

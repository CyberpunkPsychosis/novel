# 云中书斋 · 小说创作 + 阅读网站

人机协作写长篇小说的仓库，外加一个自动生成的阅读网站。作者出意图，AI 出文字；写好的章节会变成一个能在手机/电脑上读的网页书架，书越写越多。

## 目录结构

```
books/<书名>/        每本书一个文件夹，内部结构如下
  ├ book.json        书名、简介、标签、状态、封面配色（网站读取）
  ├ 0-故事核/         故事核心、主题、结局方向、红线
  ├ 1-设定库/         世界观、角色卡、时间线 —— 防人设漂移
  ├ 2-大纲/           全书大纲 + 逐章章纲 —— 防剧情崩坏
  ├ 3-文风/           文风指南 + 样章 —— 防文风不稳
  ├ 4-正文/           每章一个 .md 文件
  └ 5-一致性/         状态追踪（伏笔账本）+ 检查清单 —— 防伏笔丢失

tools/build-site.mjs  网站生成器（零依赖 Node 脚本）
docs/                 自动生成的网站（GitHub Pages 从这里发布，勿手改）
site.json             网站标题等全局配置
CLAUDE.md             AI 协作流程（写作前读档案、写作后更新档案 + 重建网站）
```

## 阅读网站

- 首页是书架：每本书一张封面卡片（封面由代码生成，可在 `book.json` 里换成自己的图）。
- 点进书 → 章节目录 → 阅读页，支持上一章/下一章、字号大小、自动深色模式。
- 网站不展示设定/大纲（那是创作区），只展示正文，避免剧透。

### 本地预览

```
node tools/build-site.mjs        # 重新生成 docs/
# 然后用浏览器打开 docs/index.html
```

### 发布到 GitHub Pages（一次性设置）

仓库 **Settings → Pages → Build and deployment**：
- Source 选 **Deploy from a branch**
- Branch 选当前分支，文件夹选 **/docs**，保存

几十秒后会给出一个网址，以后每次推送（章节 + 重建的 `docs/`）网站都会自动更新。

## 怎么写一本新书

1. 在 `books/` 下新建文件夹，照已有书的结构建好 `book.json` 和六个目录
2. 和 AI 对话填 `0-故事核`，再逐步充实设定、大纲
3. 说"写第 N 章" → AI 先给节拍简述确认 → 写正文 → 更新一致性档案 → 重建网站 → 提交
